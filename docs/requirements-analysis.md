# AI 小说转剧本工具需求分析

## 1. 项目背景

很多小说作者希望将已有作品改编成影视、短剧、广播剧或舞台剧剧本，但传统改编需要熟悉剧本格式、场景拆分、人物对白重写、节奏控制等专业能力。该项目希望通过 AI 辅助，把多章节小说文本转换为结构化剧本初稿，降低改编门槛，提高创作效率。

## 2. 项目目标

系统需要支持输入 3 个章节以上的小说文本，并自动生成 YAML 格式的结构化剧本。输出结果应当可阅读、可编辑、可校验，并方便后续扩展为剧本编辑器、分镜工具或协作改稿系统。

核心目标包括：

- 支持作者输入至少 3 个章节的小说内容。
- 自动抽取人物、地点、事件和情节线索。
- 将叙事文本改写为剧本中的场景、动作、对白、旁白和转场。
- 输出符合固定 Schema 的 YAML 剧本。
- 提供 YAML Schema 文档，说明字段定义和设计原因。
- 提供本地运行能力，并将代码托管到 GitHub。

## 3. 用户角色

### 小说作者

小说作者是主要用户。作者希望快速获得一个剧本初稿，而不是从空白文档开始改编。

### 编剧或改稿人员

编剧可以基于 YAML 结构继续修改对白、合并场景、调整人物动机和节奏。

### 项目评审或开发者

评审和开发者需要能够本地启动项目、理解接口设计、检查 Schema 合理性，并验证输出是否符合题目要求。

## 4. 功能需求

### 4.1 小说章节输入

系统应提供章节输入界面。每个章节包含章节标题和章节正文。提交生成前必须校验章节数量不少于 3 个，且每章正文不能为空。

### 4.2 AI 剧本生成

后端接收章节列表后，调用 DeepSeek API 进行剧本生成。为了提高稳定性，模型应优先输出 JSON 结构，由后端使用 Pydantic 校验后，再通过 PyYAML 转换为 YAML。

### 4.3 结构化 YAML 输出

系统输出应包含剧本元信息、原文来源、人物表、地点表、场景列表和创作备注。场景应包含时间、地点、登场人物、章节引用和剧情节拍。

### 4.4 YAML 编辑、复制和下载

前端应使用 Monaco Editor 展示生成的 YAML。用户可以编辑内容，并支持复制到剪贴板或下载为 `.yaml` 文件。

### 4.5 Schema 查看和校验

项目应提供 Markdown 格式的 YAML Schema 文档。系统后端提供 `/api/schema` 接口返回 JSON Schema，便于未来做自动校验和编辑器提示。

### 4.6 示例演示

项目应提供示例小说和示例 YAML 输出。未配置 DeepSeek API Key 时，后端允许返回示例结果，确保项目可以稳定演示。

## 5. 非功能需求

### 可用性

界面应简洁直接，作者进入页面后即可输入章节并生成剧本。生成失败时应提供明确错误提示。

### 可维护性

后端使用 Pydantic 定义数据模型，Schema、校验和输出结构保持一致。Prompt 独立存放，方便后续优化。

### 稳定性

AI 输出存在不确定性，因此系统不直接信任模型输出。后端需要解析 JSON、校验字段，并在异常时返回错误或演示数据。

### 可部署性

前端支持 GitHub Pages 静态部署。后端支持本地 FastAPI 运行，并可扩展部署到云服务。

## 6. 技术方案

### 前端

前端使用 React + Vite 构建。Tailwind CSS 负责原子化样式和响应式布局，shadcn/ui 提供按钮、表单、弹窗、提示等基础组件，motion 用于生成状态、面板切换和轻量交互动效。Monaco Editor 用于展示和编辑 YAML。axios 用于请求后端。js-yaml 用于前端 YAML 格式检查和预览扩展。

### 后端

后端使用 FastAPI 提供 REST API。Pydantic 定义请求、响应和剧本 Schema。httpx 调用 DeepSeek API。PyYAML 将校验后的 Python 对象转换为 YAML。

### AI

AI 服务使用 DeepSeek API。Prompt 要求模型输出合法 JSON，不输出解释性文字。后端负责提取 JSON、校验结构和转换 YAML。

### Schema

Schema 采用“Markdown 文档 + Pydantic 模型 + JSON Schema 思路”的组合。Markdown 说明字段含义和设计原因，Pydantic 保证运行时校验，JSON Schema 便于未来前端校验和外部系统接入。

## 7. API 设计

### GET /api/health

用于检查后端是否正常运行。

### POST /api/generate

输入小说章节，返回结构化剧本对象和 YAML 文本。

### GET /api/schema

返回由 Pydantic 模型生成的 JSON Schema。

## 8. 项目边界

当前版本重点完成“小说到结构化剧本初稿”的生成，不实现复杂协作、账号系统、在线存储、付费管理或云端后端部署。后续可扩展分镜生成、角色关系图、多版本改稿、剧集拆分和剧本质量评估。

## 9. 验收标准

- 可以输入 3 个以上章节并生成剧本。
- 输出内容为合法 YAML。
- YAML 包含 metadata、source、characters、locations、scenes、notes 等核心结构。
- 项目包含 YAML Schema 文档，并解释设计原因。
- 前端可以本地运行并展示 Monaco Editor。
- 后端可以本地运行并提供 FastAPI 接口。
- 代码已提交到 GitHub 仓库。
